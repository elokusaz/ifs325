Exercise 1: Binary Path Encoding

a.) The binary encoding for the shortest path from 1 to 18

The shortest route from node 1 to node 18 is the path 1 ‚Üí 3 ‚Üí 7 ‚Üí 13 ‚Üí 18. This path has 4 steps.

The code is found by checking if the destination node at each step is odd (0) or even (1):

1 ‚Üí 3 (odd) = 0 3 ‚Üí 7 (odd) = 0 7 ‚Üí 13 (odd) = 0 13 ‚Üí 18 (even) = 1
The final code is 0001.

b.) The binary encodings for the shortest paths that lead to 19 via 15

The shortest way to get to node 15 takes 4 steps. From there, it's one more step to node 19. This means the shortest paths to 19 that go through 15 are 5 steps long. There are four such paths:

c.) Find the binary encodings for all paths that lead from 1 to 15

There are four possible paths from node 1 to node 15. All of them are 4 steps long.

d.) What path does 0110 describe?

The code 0110 does not describe a valid path.

Here's why:
  • 0: Start at 1, go to an odd node ‚Üí 3.
  • 1: From 3, go to an even node ‚Üí 6.
  • 1: From 6, go to an even node ‚Üí 12.
  • 0: From 12, you must go to an odd node. But the only path from 12 is to node 14, which is even. This contradicts the code, so the path is impossible.

e.) Is 010001 the longest path to 18? If not, find the longest path.

No, 010001 is not the longest path to 18. It's not a valid path at all because decoding 01000 leads to node 19 (1 ‚Üí 3 ‚Üí 6 ‚Üí 11 ‚Üí 15 ‚Üí 19), which has no outgoing paths for the final digit 1.

The actual longest path to 18 has 6 steps. There are two of them:
1 ‚Üí 3 ‚Üí 7 ‚Üí 13 ‚Üí 16 ‚Üí 17 ‚Üí 18 1 ‚Üí 3 ‚Üí 6 ‚Üí 12 ‚Üí 14 ‚Üí 17 ‚Üí 18

f.) How many 5-digit paths lead to 19?

A 5-digit code means a path with 5 steps. To end at node 19, the path must first arrive at either node 15 or node 18 in 4 steps.

Paths via Node 15: 4 unique paths from 1 to 15 that take 4 steps Paths via Node 18: 1 unique path from 1 to 18 that takes 4 steps Total paths = 4 + 1 = 5
There are five 5-digit paths that lead to node 19.

Exercise 2: Trinary Tree Paths

a.) How to encode the decisions at each point

To encode the decisions at each point, you can use the trinary digits {0, 1, 2} to represent the three possible routes. Since there are three equally likely paths at each fork (left, middle, right), a natural mapping would be:

0 for the left path 1 for the middle path 2 for the right path

A complete path from the root (point 1) to any endpoint can thus be represented by a unique sequence of these three digits.

b.) Find log‚ÇÉ(j) for the end of any path in this tree

In this context, j represents the total number of endpoints in the tree. We can see that after the first level of forks there are 3 nodes, after the second there are 3√ó3=9 nodes, and at the end (after the third level of forks), there are 9√ó3=27 endpoints.

So, j = 27 The value of log‚ÇÉ(j) is: log‚ÇÉ(27) Since 3¬≥ = 27, the result is: log‚ÇÉ(27) = 3
This value, 3, represents the number of decisions or levels of forks a path must traverse to get from the root to an endpoint.

c.) Choose a path from 1 to 3 and give its trinary encoding

Let's choose a path that starts by going from point 1 to 3 and then continues to an endpoint. From point 3, we'll take the right path, and from the subsequent point, we'll take the middle path.

Path Description:
  • Start at point 1.
  • Take the middle branch to point 3.
  • From point 3, take the right branch.
  • From that new point, take the middle branch to reach an endpoint.
Point Numbers: The only numbered points on this path are 1 and 3. Trinary Encoding: Using the scheme from part (a): - The first decision (from 1 to 3) is the middle path, which is encoded as 1. - The second decision is the right path, encoded as 2. - The third decision is the middle path, encoded as 1.
Therefore, the trinary encoding for this specific path is 122.

d.) How many endpoints would there be if there were 5 levels of forks instead of 3?

The number of endpoints in this type of tree is calculated by raising the branching factor (3) to the power of the number of levels of forks.

For 3 levels of forks, the number of endpoints is 3¬≥ = 27. If there were 5 levels of forks, the number of endpoints would be: 3‚Åµ = 3√ó3√ó3√ó3√ó3 = 243
There would be 243 endpoints. ‚úÖ
